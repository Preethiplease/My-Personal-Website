A university project. Developed in Atom IDE using HTML, CSS and JavaScript.

A music recommendation Web application using a cloud-client (server) architecture developed deploying Agile Methodology. Google cloud Firestore database has been used to store the data.

Implementation of data analytics algorithms with machine learning i.e., Distance classifiers such as Euclidean Distance, Manhattan Distance and RBF to perform the recommendations for the user. The results are viewed in a PDF format to the user.

The 4 sections in first page of the PDF are:
  • The Recommendation system's logo
  • The graph of 4 different emotion category the songs fall in
  • The location of the Album's origin and
  • The songs in the database followed by the results generated by the 3 algorithms based on user preference.

Project 3 - Mobile Robots

The Initial Layout
Finding the beacon
Working of PID controller
❮ ❯
A university project. Programming of a mobile robot (pioneer 3D-X) in Coppeliasim (V-Rep) using Python.

The robot had to complete several tasks:
  • Task 1: find the middle of the starting room and exit
  • Task 2: find the beacon
  • Task 3: map the environment to a certain degree
  • Task 4: return to the centre of the starting room

In the end the robot the robot has a Finite State Machine (FSM) to make sure that the tasks are completed.

It uses ultrasonic proximity sensors to avoid obstacles and to map the environment on an occupancy grid. It uses a PID controller to keep inline with the path or to follow walls.

Project 4 - Solving TSP with K-NN algorithm

Unity Workspace

❮ ❯
A university project. Developed using Unity, coded in C#.

Implementation of K-Nearest neighbor algorithm to find the shortest path to solve the travelling salesman problem.

The algorithm in 4 steps:
  • Start with a random initial node, call in the ‚Äòcurrent node‚Äô (start city).
  • Find the nearest unvisited node as the next point to travel to.
  • Now set this node as ‚Äòcurrent node‚Äô and mark it as ‚Äòvisited‚Äô, and find the next unvisited nearest node.
  • When all the nodes are visited once, terminate the condition and return to the starting node.

Project 5 - Natural Language Processing

The resulting chatbot response
The reduction of loss as iterations increased
❮ ❯
A university project. Developed using Jupiter Python, scripted in Python.

The main purpose of this project is to make possible modifications to a basic recurrent sequence-to-sequence chatbot code developed by Matthew Inka, provided through a tutorial link with different parameters to tune in such a way that the chatbot can interact with the user much efficiently with appropriate responses for sequenced questions.

The tutorial code given is trained with 4000 words in English with lines from the cornel movie and 500 iterations to train the chatbot with.

The chatbot was able to respond to, and identify the vocabulary within the Cornell movie dataset but it displayed the message ‚ÄòUnknown word‚Äô every time it encountered new words beyond it.

Considering this as a drawback, I have trained the robot multiple times by increasing the iterations to 5000 and the chatbot‚Äôs responses have improved.
